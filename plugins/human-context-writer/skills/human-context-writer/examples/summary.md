# Summary Example

## User request

Summarize this:

"The new plugin system allows agents to discover, install, and manage skills from a centralized marketplace. Instead of manually configuring each skill, users can search for what they need and install it in one command. The system validates plugin metadata before installation, checks for conflicts with existing skills, and generates a summary report after each install session. This reduces setup time significantly and makes agent configuration more predictable."

## Weak AI-style summary

The text discusses a new plugin system that provides various benefits for agents and users, including improved efficiency and a more streamlined approach to skill management and configuration.

## Human-style summary

The new plugin system lets agents find and install skills from a marketplace with one command. It validates metadata, checks for conflicts, and shows a summary after each install. Setup is faster and more predictable.

## Notes

A good summary should:

- keep the key points without inflating them
- remove filler like "various benefits" and "the text discusses"
- not add meaning that was not in the original
- match the length to what is actually needed, not pad it
