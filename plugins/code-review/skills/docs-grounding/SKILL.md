# Docs Grounding

Use this whenever making a framework/tool claim. Without MCP, use project-local docs, official docs via built-in browsing/search if available, official repo docs, language/runtime docs, OWASP, MDN, W3C, RFCs, and database vendor docs.

If documentation cannot be verified, label the claim as analysis or Unverified.
