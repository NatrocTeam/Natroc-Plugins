# Audit Principles

Core rules:

- No evidence, no finding.
- No official docs or authoritative source, no hard framework claim.
- No file reference, no code-specific accusation.
- No attack path or abuse case, no security severity above Unverified.
- No automatic fix without explicit user approval.
- No PASS if verification failed or was not run.

Separate documented fact, authoritative best practice, and technical analysis.
